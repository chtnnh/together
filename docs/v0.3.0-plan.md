## Existing feature: profanity filter (already implemented)

Server-side in `services/realtime/src/room-do.ts`. Host toggles in room settings ‚Üí Chat ‚Üí ‚ÄúProfanity filter‚Äù (default off). On each message, `filterProfanity()` replaces words from `PROFANITY_WORDS` in `packages/shared/src/constants.ts` with asterisks before broadcast. Messages are censored, not blocked.

## Phase 1 ‚Äî Fix stale participant count (P0 bug)

**Root cause:** `room-do.ts` persists full `RoomState` (including `participants[]`) on queue/chat/playback updates, but not on disconnect. After DO hibernation, stored ghosts reload.

**Fix:**

1. Stop persisting `participants` and `chat` in `persist()` ‚Äî ephemeral only
2. Reconcile on wake ‚Äî clear `participants` and `chat` after storage load
3. Implement `leave` event + ensure close handler removes participants
4. `/stats` uses live session count (unique participantIds)

**Files:** `services/realtime/src/room-do.ts`

**Acceptance:** Everyone leaves ‚Üí one user rejoins later ‚Üí count shows `1`.

## Phase 2 ‚Äî DO lifecycle + Supabase persistence

### 2.1 Empty-room alarm + state trimming

| Trigger | Action |
|---------|--------|
| Last WebSocket closes | `setAlarm(now + EMPTY_ROOM_GRACE_MS)` (default 5 min) |
| Alarm fires, 0 sessions | Snapshot queue/playback to Postgres; clear DO storage |
| New join before alarm | Cancel alarm |

### 2.2 Room snapshot in Postgres

Add `live_snapshot` jsonb + `last_active_at` timestamptz on `rooms`. Internal API `POST /api/internal/rooms/[slug]/snapshot`. Hydrate on room init when DO is cold.

### 2.3 Chat ephemeral UX

- Join-only `chat:notice` event (not stored in history)
- Zero state in `emoji-chat.tsx` when no messages

### 2.4 Wire dormant schema

- `room_bans` ‚Äî persist bans to Postgres; hydrate on DO init
- `users.app_role` ‚Äî `user` \| `superadmin` (Phase 7)
